# Закупка, ремонт и координация планет

## 1. Разделение ответственности

```text
Штаб:
"Нужно усилить защиту от истребителей."

Закупка:
"Купить корвет на планете №3 за 25 кредитов."
```

Штаб формирует потребность, закупщик выбирает конкретное действие.

## 2. Директива штаба

```json
{
  "goal": "COUNTER_FAST_INTERCEPTORS",
  "maxSpend": 35,
  "minimumReserve": 10,
  "desiredFleetChanges": {
    "ANTI_INTERCEPTOR_CORVETTE": 1,
    "COLONY_SHIP": 0
  },
  "avoidPlanetIds": [2],
  "preferredPlanetIds": [3]
}
```

## 3. Что получает закупщик

- кредиты;
- ожидаемый доход;
- состав обоих флотов;
- свои планеты;
- угрозы рядом;
- занятость клеток;
- список допустимых покупок;
- ремонтные намерения штаба.

## 4. Ремонтные намерения

```json
{
  "repairIntentions": [
    {
      "unitId": 7,
      "planetId": 3
    }
  ]
}
```

## 5. Почему это важно

Покупка может занять клетку, заблокировать вход повреждённому кораблю, создать пробку или сделать ремонтную планету уязвимой. Поэтому закупщик должен учитывать будущие прибытия.

## 6. Ограничения

- не более одной покупки на одной планете за ход;
- не превышать `maxSpend`;
- не опускаться ниже `minimumReserve`;
- не покупать колонизатор без нейтральной цели;
- не блокировать критический ремонт;
- разрешить не покупать ничего;
- повторно проверить покупку перед исполнением.

## 7. Порядок покупки и ремонта

Нужно зафиксировать конкретную механику, например:

```text
закупка
→ движение
→ ремонт в конце хода
```

или другой вариант. AI-контракт должен соответствовать реальным правилам игры.

## 8. Безопасное поведение при ошибке

Если агент закупок не ответил или выбрал неверный ID, лучше сохранить кредиты:

```json
{
  "purchaseActionIds": [],
  "reasonCode": "AGENT_ERROR_SAVE_CREDITS"
}
```
