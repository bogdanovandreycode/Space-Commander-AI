# Конвейер AI-хода

## 1. Полная последовательность

```text
1. Зафиксировать начало AI-хода
2. Построить глобальное состояние
3. Запросить план штаба
4. Валидировать план штаба
5. Запустить агента закупок
6. Валидировать и исполнить покупки
7. Нормализовать очередность юнитов
8. Для каждого юнита:
   8.1. Проверить, что юнит существует и может ходить
   8.2. Построить локальный контекст
   8.3. Сгенерировать legalActions
   8.4. Запросить решение
   8.5. Проверить actionId
   8.6. Повторно пересчитать legalActions
   8.7. Исполнить действие
   8.8. Записать событие
   8.9. Сформировать рапорт
9. Проверить победу
10. Завершить AI-ход
```

## 2. Этап штаба

Штаб получает только текущий ход:

- номер хода;
- размер карты;
- все юниты и планеты;
- кредиты и доход;
- компактные правила классов;
- доступные оперативные возможности;
- состояние ремонта и занятости планет.

История предыдущих ходов по умолчанию не нужна.

Штаб возвращает:

- доктрину;
- приоритеты;
- рекомендации;
- порядок юнитов;
- директиву закупки;
- короткий комментарий.

## 3. Этап закупки

Закупщик должен учитывать:

- планеты, куда штаб направляет корабли на ремонт;
- занятость клеток;
- минимальный резерв;
- состав противника;
- наличие нейтральных целей;
- угрозы планетам.

После каждой покупки список допустимых покупок пересчитывается.

## 4. Этап юнита

### 4.1. Локальная область

В локальную область входят:

- объекты в радиусе 3–4 клеток;
- клетки, доступные юниту сейчас;
- враги, способные атаковать его следующим ходом;
- ближайшие союзники;
- свои планеты;
- ремонтные точки;
- риск каждой доступной позиции.

### 4.2. Решение

Пример ответа:

```json
{
  "actionId": 16,
  "recommendationStatus": "DEFERRED_UNSAFE",
  "intentCode": "REPOSITION_FOR_NEXT_TURN",
  "reasonCode": "LETHAL_COUNTERATTACK",
  "confidence": 0.84
}
```

### 4.3. Повторная проверка

Перед исполнением нужно заново получить объект юнита и пересчитать `legalActions`. Если выбранное действие исчезло, применяется безопасный fallback.

## 5. Рапорт

Рапорт создаётся только после `actualResult`:

```text
решение
→ валидация
→ исполнение
→ фактический результат
→ рапорт
```

## 6. Fallback

- штаб не ответил — старый AI или стандартный план;
- закупка не ответила — не покупать;
- юнит не ответил — выбрать лучший движковый score;
- рапорт не ответил — пропустить;
- критическая ошибка — разблокировать UI и завершить ход безопасно.

## 7. Параллельность

Решения кораблей нельзя безопасно исполнять параллельно. Допустимо асинхронно генерировать рапорт предыдущего юнита, но следующий тактический запрос должен использовать актуальную карту.
