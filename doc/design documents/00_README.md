# Civ3 AI — документация и техническое задание

## Назначение

Этот комплект документов фиксирует итоговую концепцию AI для небольшой пошаговой стратегии `civ3`.

Основная идея: отказаться от одного общего AI-запроса, который одновременно пытается быть стратегом, тактиком, экономистом и комментатором. Вместо этого AI делится на специализированные роли:

1. **Штаб** — анализирует полную карту, определяет цели, приоритеты и порядок действий.
2. **Агент закупок** — решает, что строить и где, с учётом экономики, угроз и планов ремонта.
3. **Агенты отдельных юнитов** — получают локальную тактическую обстановку и рекомендацию штаба, после чего выбирают конкретное действие.
4. **Агенты рапортов** — описывают фактический результат действий отдельных кораблей.

LLM никогда не должна напрямую изменять состояние игры. JavaScript-движок остаётся единственным источником истины и отвечает за допустимость движения, урон, броню, бонусы против классов, уничтожение, колонизацию, ремонт, кулдауны, занятость клеток, победу и поражение.

## Ключевой принцип

```text
LLM выбирает намерение или actionId
→ JavaScript повторно проверяет действие
→ JavaScript исполняет действие
→ LLM получает фактический результат
→ LLM формирует рапорт
```

## Основные решения

1. Полная история диалога с LLM не хранится.
2. Каждый запрос самодостаточен.
3. Штаб видит всю карту.
4. Отдельный юнит видит локальную тактическую область.
5. Штаб выдаёт рекомендации, а не абсолютные приказы.
6. Штаб определяет порядок обработки юнитов.
7. Юниты ходят последовательно по обновляемой карте.
8. JavaScript формирует и проверяет `legalActions`.
9. Закупка выделена в отдельного агента.
10. Рапорты не влияют на механику.
11. Для штаба можно использовать более сильную локальную или облачную модель.
12. Для кораблей можно использовать небольшую локальную модель, например Gemma 4B.
13. Реальные названия юнитов не должны быть единственным источником смысла.
14. Для LLM используются семантические классы и компактные правила.
15. Старый AI желательно сохранить как fallback.

## Масштаб контекста

Даже при карте `10×10` пустые клетки не передаются. В редком пиковом сценарии ожидается около 40 юнитов на двух игроков и 15–20 планет, то есть примерно 55–60 занятых клеток. Контекст не растёт по ходам: его размер ограничен текущим состоянием карты, правилами и числом доступных действий.
